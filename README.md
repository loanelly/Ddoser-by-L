# Async Web Performance & WAF Auditor

**Async Web Performance & WAF Auditor** — это учебный консольный инструмент на Python, разработанный для проведения асинхронного нагрузочного тестирования веб-ресурсов, анализа стабильности веб-серверов и верификации механизмов защиты (WAF / Rate Limiting).

Проект создан исключительно в образовательных целях для демонстрации принципов конкурентности (Concurrency) в Python и методов анализа защищенности инфраструктуры (White Hat Auditing).

---

## 🚀 Основной функционал

*   **Высокая производительность (`asyncio` + `aiohttp`)**: Использование асинхронной архитектуры вместо тяжелых системных потоков позволяет генерировать контролируемую конкурентную нагрузку с минимальным потреблением ресурсов RAM и CPU.
*   **Интерактивный интерфейс**: Динамический ввод и валидация целевого домена непосредственно при запуске скрипта через монитор порта.
*   **Имитация пользовательского поведения**: Автоматическая рандомизация заголовков `User-Agent` из встроенного пула для предотвращения ложных блокировок по сигнатуре скрипта.
*   **Адаптивный обход ограничений (Smart Rate Limiting)**: Скрипт анализирует ответы сервера. При фиксации HTTP-статуса `429 (Too Many Requests)` активируется алгоритм экспоненциальной паузы (`asyncio.sleep`), снижающий частоту отправки для завершения сессии аудита.
*   **Автоматическое логирование в CSV**: Все метрики (ID запроса, HTTP-статус, задержка ответа `Latency`, задействованный `User-Agent`) логируются в реальном времени и экспортируются в файл `audit_report.csv` для последующего анализа в Excel.

---

## 🛠 Архитектура инструмента

Программа реализует паттерн работы с семафорами (`asyncio.Semaphore`), что гарантирует строгое соблюдение установленных лимитов конкурентности. В отличие от примитивных утилит (например, `ping`), данный инструмент работает на прикладном уровне (L7 модели OSI), анализируя полноценный цикл HTTP-сессий.

## 📋 Требования и установка

Для работы скрипта требуется интерпретатор **Python 3.8+** и установленные сетевые зависимости.

1. Клонируйте репозиторий:
```bash
git clone https://github.com/loanelly/Ddoser-by-L
cd async-waf-auditor
```

2. Установите необходимые библиотеки:
```bash
pip install aiohttp requests
```

---

## 💻 Использование

Запустите главный скрипт:
```bash
python DDoserL1.1.0.py
```

После запуска утилита предложит ввести целевой домен:
```text
============================================================
 АСИНХРОННЫЙ АУДИТОР СЕТЕВОЙ БЕЗОПАСНОСТИ И ПРОИЗВОДИТЕЛЬНОСТИ
============================================================
Введите целевой домен (например, ddostest.me): ddostest.me
```

По окончании работы скрипт выведет:
1. Минимальное, среднее и максимальное время задержки (Latency).
2. Сводную таблицу полученных HTTP-статусов.
3. Экспертное заключение о наличии защиты (Rate Limiting / Web Application Firewall).
4. Файл `audit_report.csv` в корневой директории.

---

## 📊 Формат выходных данных (CSV)

Сгенерированный файл `audit_report.csv` содержит следующие поля:
*   `Request_ID` — порядковый номер запроса.
*   `Status` — HTTP-код ответа сервера (или `TIMEOUT`/`ERROR` при сбоях).
*   `Latency_Sec` — точное время обработки запроса сервером в секундах.
*   `User_Agent` — строка идентификатора, использованная при запросе.
*   `Error` — тип ошибки при её возникновении.

Данный файл готов для импорта в BI-системы или Excel для построения графиков деградации производительности сервера под нагрузкой.
